# Design options comparison

Early in a project the design is not settled. There may be two structural schemes, three facade treatments, or a choice between an in-situ and a precast approach. Design options let you price each variant properly and compare them on the numbers that matter, rather than on a gut feeling.

## What it is for

The workflow gives each design variant its own priced bill of quantities, then weighs the full set of options against each other. You compare them on total cost, on the difference trade by trade, and on cost per square metre, so you can see not just which option is cheaper overall but where the money actually moves between them. When you are ready to commit, you pick a baseline and get a recommendation, checked for fairness so the comparison is not skewed by one option being estimated more completely than another.

## When to use it

Reach for design options whenever a real decision hangs on cost and more than one design could satisfy the brief. It suits value engineering, scheme selection at concept or feasibility stage, and any moment where a client asks what the alternative would cost. It is most useful before the design is frozen, when changing course is still cheap.

## The steps

1. Create the options for a project. Each option pairs a source, a model or a document, with its own bill of quantities, so the variants stay separate and fully priced.
2. Build or import the priced BOQ for each option, using the same [takeoff](./quantity-takeoff.md), [matching](./bim-to-cost-and-carbon.md) and [cost base](./world-cost-bases.md) tools you would use for any estimate. Pricing every option the same way is what makes the comparison fair.
3. Open the comparison. Read the options side by side on total cost, on the by-trade deltas that show where they differ, and on cost per square metre for a size-normalised view.
4. Choose a baseline and review the recommendation. The fairness check flags where one option is carrying scope another is missing, so you can close the gap before deciding.

## How it connects

Design options build on the same [bill of quantities](./estimating-and-boq.md) and [cost bases](./world-cost-bases.md) as any estimate, and they reuse the [BIM hub](./bim-to-cost-and-carbon.md), element matching, the 5D cost model, reporting and tendering when those are installed. Once an option is chosen, its BOQ becomes the project estimate and flows on into [planning and cost control](./planning-and-cost-control.md) and, when the time comes, into a [tender](./tendering-and-bids.md).
